## 💡 카운터 앱 동작 원리 4단계 분석

이 코드는 크게 **1) 라이브러리 로드**, **2) 디자인(CSS)**, **3) 화면 뼈대(HTML)**, **4) 동작 로직(JS)** 네 부분으로 나뉩니다.

---

### ① Vue 사용 허가받기 (CDN 로드)
```html
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
```
* **설명**: 인터넷(CDN)을 통해 Vue 3의 핵심 기능을 불러오는 코드입니다. 이 코드 덕분에 브라우저가 아래에서 쓸 `Vue`라는 단어와 문법을 알아들을 수 있게 됩니다.

---

### ② 화면 그리기 (HTML 템플릿)
```html
<div id="app">
  <div class="counter-card">
    <h1>Vue 3 카운터</h1>
    
    <!-- 1. count 변수의 값을 화면에 보여주는 구멍 -->
    <div class="display" :class="{ negative: count < 0 }">
      {{ count }}
    </div>

    <div class="button-group">
      <!-- 2. 클릭하면 decrease, increase 함수를 실행하라는 명령 -->
      <button class="btn-minus" @click="decrease">-</button>
      <button @click="increase">+</button>
    </div>
  </div>
</div>
```
* `id="app"`: Vue에게 **"앞으로 이 div 태그 안쪽 공간을 네가 관리해라"** 하고 영역을 지정해 준 것입니다.
* `{{ count }}`: **머스태시(Mustache) 문법**이라고 합니다. 자바스크립트 안의 `count`라는 변수의 값을 실시간으로 이곳에 보여줍니다.
* `@click="increase"`: **이벤트 바인딩**입니다. "이 버튼을 **클릭(`@click`)**하면 자바스크립트에 만들어 둔 `increase` 함수를 실행해라"라는 뜻입니다. (원래는 `v-on:click`이지만 줄여서 `@click`을 주로 씁니다.)
* `:class="{ negative: count < 0 }"`: **클래스 바인딩**이라는 아주 강력한 기능입니다. 
  * `count`가 0보다 작아지면(`count < 0`이 참이 되면) 자동으로 `negative`라는 CSS 클래스를 이 태그에 갖다 붙여줍니다. 
  * 덕분에 숫자가 음수가 되면 숫자가 빨간색으로 스르륵 변하게 됩니다.

### ③ 동작 조작하기 (JavaScript)
```javascript
// 1. Vue 상자에서 필요한 도구(createApp, ref)를 꺼냅니다.
const { createApp, ref } = Vue;

createApp({
  // 2. Vue 앱의 설정 및 데이터/기능을 조립하는 공간입니다.
  setup() {
    
    // 3. '반응형 변수'를 선언합니다.
    const count = ref(0);

    // 4. 숫자를 1 더하는 함수
    const increase = () => {
      count.value = count.value + 1;
    };

    // 5. 숫자를 1 빼는 함수
    const decrease = () => {
      count.value = count.value - 1;
    };

    // 6. 위에서 만든 변수와 함수를 바깥 HTML 화면으로 전달(return)합니다.
    return {
      count,
      increase,
      decrease
    }
  }
}).mount('#app'); // 7. 조립한 Vue 앱을 HTML 상의 #app 구역에 결합(마운트)합니다.
```

## 🔍 핵심 개념 요약

### 1. `ref(0)` (반응형 변수)
* 일반 자바스크립트 변수(`let count = 0`)는 값이 1, 2, 3으로 변해도 **웹 화면이 알아채지 못해 화면은 계속 0으로 멈춰 있습니다.**
* `ref(0)`을 사용하면 Vue에게 **"이 변수는 감시 대상이야. 값이 변하면 이 값을 쓰고 있는 화면(`{{ count }}`)도 실시간으로 다시 그려줘!"** 하고 신청을 하는 것입니다. 이를 **반응성(Reactivity)**이라고 합니다.

### 2. `.value` 의 비밀
* 자바스크립트 코드 안에서 `ref`로 만든 변수를 읽거나 수정할 때는 반드시 뒤에 `.value`를 붙여야 합니다 (`count.value = count.value + 1`).
* 왜냐하면 `ref`는 단순히 값만 담고 있는 게 아니라, "값이 바뀌는 것을 감시하는 감시 센서"가 달린 객체이기 때문입니다. 그 안의 진짜 값을 꺼내기 위해 `.value`를 씁니다.
* **단, HTML 영역(`{{ count }}`) 안에서는 Vue가 자동으로 `.value`를 떼어내서 보여주기 때문에 쓰지 않아도 됩니다!**
