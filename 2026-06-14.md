## 💡 1. 왜 CDN에서 'Vite 빌드 환경'으로 넘어왔을까요?

1단계에서는 HTML 파일 하나에 CDN으로 Vue를 불러와서 실습했습니다. 하지만 웹사이트가 커지고 기능이 많아지면 HTML 파일 하나에 모든 코드를 담기가 어려워집니다.

* **CDN 방식의 한계**: 파일이 거대해지고 관리가 불가능해짐, CSS가 꼬임, 최신 자바스크립트 문법(SFC 등)을 브라우저가 직접 실행하지 못함.
* **현대적인 도구(Vite + npm) 도입**:
  * **npm (Node Package Manager)**: 스마트폰의 앱스토어처럼, 필요한 외부 라이브러리(Vue 등)를 컴퓨터에 다운로드하고 관리해 주는 도구입니다. `package.json`은 이 라이브러리들의 목록(영수증)입니다.
  * **Vite (비트)**: 아주 빠른 개발용 서버를 띄워주고, 여러 개로 쪼개진 코드 파일들을 브라우저가 읽을 수 있는 최적의 상태로 조립(빌드)해 주는 현대적인 도구입니다.

---

## 📂 2. 싱글 파일 컴포넌트 (SFC: Single File Component) 란?

SFC는 **"하나의 파일(`.vue`) 안에 HTML, JavaScript, CSS를 모두 담아 하나의 독립된 부품(컴포넌트)을 만드는 것"**을 의미합니다.

Vue 파일은 크게 다음과 같은 3단 구조로 나뉩니다.

```vue
<script setup>
/* ① 자바스크립트 영역: 데이터와 동작 로직 정의 */
</script>

<template>
<!-- ② HTML 영역: 화면의 모양(뼈대) 설계 -->
</template>

<style scoped>
/* ③ CSS 영역: 화면의 스타일(디자인) 꾸미기 */
</style>
```

이렇게 하나의 단위로 쪼개두면, 나중에 필요한 화면에 이 컴포넌트를 부품처럼 가져와 조립할 수 있어 재사용성과 관리가 극대화됩니다.

## 🔍 3. 오늘 작성한 App.vue 코드 해부하기

오늘 작성한 [App.vue](file:///d:/my-vue-study/src/App.vue) 파일의 구조를 부분별로 쪼개서 살펴보겠습니다.

### ① `<script setup>` (동작 로직 - JS)

```javascript
import { ref } from 'vue' // 1. Vue에서 반응성 도구 가져오기

const count = ref(0) // 2. 반응형 변수 count 정의 (기본값 0)

const increment = () => { // 3. 클릭 시 실행할 함수 정의
  count.value++
}
```

* **`<script setup>`의 마법**: 
  * 1단계에서는 `setup()` 함수를 작성하고 마지막에 `return { count, increase }`처럼 밖으로 보낼 변수들을 일일이 적어주어야 했습니다.
  * 하지만 `setup` 속성을 쓰면, **선언한 변수와 함수가 자동으로 HTML 템플릿 영역으로 전달**됩니다! 훨씬 코드가 간결하고 직관적입니다.
* **`count.value++`**: 
  * 자바스크립트 영역에서 `ref` 값을 변경할 때는 꼭 `.value`를 붙여야 합니다. `count.value = count.value + 1`을 짧게 쓴 것입니다.

---

### ② `<template>` (화면 구성 - HTML)

```html
<template>
  <div class="container">
    <h1>Vue SFC & Composition API 🌟</h1>
    <!-- 버튼을 누르면 increment 함수 실행, 화면에는 실시간 count 변수 출력 -->
    <button @click="increment">카운트: {{ count }}</button>
  </div>
</template>
```

* **`@click="increment"`**: 마우스 클릭 이벤트가 발생하면 위에 선언한 `increment` 함수를 실행합니다.
* **`{{ count }}`**: 반응형 변수의 변화를 Vue가 실시간 감시하고 있다가, 버튼 클릭으로 `count.value`가 바뀌면 **화면의 이 숫자를 알아서 변경**해 줍니다. 
  * HTML 영역에서는 `.value`를 생략하고 `count` 이름만 쓰면 됩니다.

---

### ③ `<style scoped>` (디자인 - CSS)

```css
<style scoped>
.container {
  max-width: 600px;
  margin: 50px auto;
  text-align: center;
}
button {
  background-color: #42b883; /* Vue 공식 초록색 */
  color: white;
  border: none;
  padding: 10px 20px;
  border-radius: 4px;
  cursor: pointer;
}
button:hover {
  background-color: #35495e; /* 마우스 올렸을 때 남색으로 변함 */
}
</style>
```

* **`scoped` (스코프드)의 엄청난 이점**:
  * 그냥 `<style>`을 쓰면 이 CSS 스타일이 사이트 전체의 모든 버튼에 영향을 미치게 됩니다.
  * 하지만 `scoped`를 붙여주면 **"오직 이 파일(App.vue) 안에 있는 태그에만 이 스타일을 적용하겠다"**라는 뜻이 되어, 다른 컴포넌트의 디자인을 침범하지 않고 독립성을 유지할 수 있습니다.
